# Swarm‑IDE 开发备忘

> 本文件记录了项目的整体架构、核心技术实现与关键设计思路，方便后续维护和快速上手。

## 1️⃣ 项目结构概览
```
.
├─ assets/                     # 静态资源（视频、图片、二维码）
├─ backend/                    # 后端核心代码

│   │   ├─ _components/       # React 组件
│   │   ├─ api/                # Next.js API 路由
│   │   ├─ graph/              # 拓扑渲染相关
│   │   ├─ im/                 # WebSocket / SSE 实现

│   │   └─ globals.css         # Tailwind + 全局样式
│   ├─ config/app.json          # 数据库等配置
│   ├─ scripts/query.mjs       # 工具脚本
│   ├─ skills/                 # 技能目录（Markdown）
│   │   └── topology/

│   │   ├─ db/                  # ORM / 数据库模型
│   │   ├─ lib/                 # 通用工具（LLM、MCP 等）
│   │   └─ runtime/             # Agent 运行时管理
│   ├─ mcp.json                # MCP 配置
│   ├─ next.config.*           # Next.js 配置
│   └─ package.json
├─ promo-video/
├─ specs/
├─ spells/

```

## 2️⃣ 核心技术栈
- **后端**：Bun + Next.js API（ESM）
- **前端**：React 18 + TypeScript + TailwindCSS + D3/Vis.js
- **数据库**：SQLite/PostgreSQL（Prisma 或自定义 ORM）
- **LLM 接口**：OpenRouter、GLM 等，配置在 `.env.local`
- **Agent Runtime**：`src/runtime` 负责 Agent 生命周期、消息队列与 Skill 注入
- **MCP**：多通道协议（stdio / http / sse），通过 `mcp.json` 自动加载

## 3️⃣ 后端核心模块
| 模块 | 主要职责 |
|------|----------|
| **Agent Runtime (`src/runtime`)** | 管理 Agent 实例、原语 `create/send/receive`、消息队列、Skill 注入、MCP 通信 |
| **Skill Engine (`skills/`)** | 自动扫描 Markdown，读取 `auto-load: true` 前置信息并注入新 Agent |
| **MCP Loader (`mcp.json`)** | 定义多种后端服务，统一 `sendMessage()` 接口 |
| **API 路由 (`app/api`)** | `/admin/init-db`、`/agents/*` CRUD 与消息发送、`/skills/*` 查询 |
| **Graph Service (`app/graph`)** | 生成拓扑数据并通过 WebSocket/SSE 推送给前端 |
| **IM Layer (`app/im`)** | WebSocket / SSE 服务器，推送 Agent 消息与状态变更 |

## 4️⃣ 前端实现
- **页面结构**：`app/page.tsx` 包含 Sidebar（对话列表）和 MainArea（Graph + ChatPanel）。
- **Graph 渲染**：使用 D3 force simulation，实时订阅 `/api/im/graph`。
- **Chat Panel**：通过 SSE/WebSocket 监听指定 Agent 的消息流，支持发送文本。

## 5️⃣ 数据库与持久化
- 表结构示例：`agents(id, name, parent_id)`, `messages(from_agent, to_agent, body)`, `skills(name, content, auto_load)`。
- 初始化接口 `/api/admin/init-db`，一次性执行迁移脚本。

## 6️⃣ 环境变量 & 配置
| Variable | 用途 |
|----------|------|
| `GLM_API_KEY`, `GLM_MODEL` | GLM 调用 |
| `OPENROUTER_API_KEY`, `OPENROUTER_MODEL` | OpenRouter 调用 |
| `DATABASE_URL` | 数据库连接 |
| `AGENT_SKILLS_DIR` | 自定义技能目录 |
| `MCP_CONFIG_PATH` | 指定 MCP 配置文件 |

## 7️⃣ Docker & 开发流程
```bash
# 启动后端 + DB
docker compose up -d
# 初始化数据库（一次性）
curl -X POST http://127.0.0.1:3017/api/admin/init-db
# 安装依赖 & 开发服务器
cd backend
bun install
bun dev
```
访问 `http://localhost:3017`。

## 8️⃣ 技术亮点
- 极简原语（create + send）
- 液态拓扑：Agent 可随时雇佣子代理
- 扁平协作 + 微信式 UI，任意层级可聊天
- MCP 多通道协议，支持本地脚本、HTTP、SSE
- Skill 自动注入（Markdown 前置信息）
- 实时 Graph 与 SSE/WebSocket 推送

## 9️⃣ 如何扩展
1. **新增 Skill**：在 `backend/skills/` 新建 Markdown 并写 `auto-load: true`。
2. **更换 LLM**：修改 `.env.local` 中的 API key / model 名称。
3. **添加 MCP 服务器**：在 `mcp.json` 新增条目，后端会自动启动。
4. **自定义数据库**：更新 ORM schema 并重新运行迁移。
